Правила HTML

HTML (HyperText Markup Language) - это код, который используется для структурирования и отображения веб-страницы и её контента. Например, контент может быть структурирован внутри множества параграфов, маркированных списков или с использованием изображений и таблиц данных.

HTML не является языком программирования - это язык разметки. В переводе с английского HyperText Markup Language расшифровывается как (язык разметки гипертекста) и используется, чтобы сообщать вашему браузеру, как отображать веб-страницы, которые вы посещаете.

  • язык означает, что он может быть прочитан, как человеком, так и компьютером;
  • разметка означает, что написанный вами код помечается с помощью ключевых слов;
  • гипертекст означает, что он использует HTTP как часть Интернета.

Простыми словами, HTML задаёт структуру содержимого и его смысл, определяя такой контент как, к примеру, заголовки, абзацы или изображения.

Как и любой язык, HTML поставляется с набором правил. Эти правила относительно простые и сводятся к определению границ, чтобы знать, где что-то начинается и где заканчивается.

Теги и элементы

Если мы посмотрим на код HTML, то увидим, что помимо самого текста, который выводится в браузере, код содержит несколько команд в угловых скобках (к примеру, <h1>). Такие команды называются элементами HTML. и состоят из нескольких частей: открывающего тега, содержимого и закрывающего тега

Элемент - это часть веб-страницы. В HTML элемент может содержать данные, фрагмент текста или изображения, или не содержать ничего. Обычный элемент включает в себя открывающий тэг, содержимое и закрывающий тэг.

Элементы и тэги это не одни и те же вещи. В HTML теги используются для создания элементов. Тэги открывают или закрывают элементы в исходном коде, тогда как элементы указывают браузеру, как отображать содержимое на странице.

Имя HTML-элемента - это имя заключённое в угловые скобки. Добавление угловые скобок < > вокруг имени элемента создаёт то, что известно как тег. Каждый из тегов несёт определённый смысл. В нашем случае <h1> обозначает Заголовок текста.

Парные теги

Как правило, теги идут парами

  • Открывающий тег показывает начало элемента. Он состоит из символа <, затем идёт имя элемента и завершается символом >; например, <h1>.
  • Закрывающий тег отмечает конец элемента. Он состоит из символа < с последующей косой чертой и именем элемента и завершается символом >; например, </h1>.

При объединении открывающего, закрывающего тегов и всего между ними, вы получите элемент HTML. Строка целиком представляет собой элемент HTML, который использует теги HTML <h1> и </h1>.

Содержимое, которое находится между открывающим и закрывающим тегами, является содержимым этого элемента. По закрывающему тегу мы определяем, что элемент завершён.

Если вы просмотрите этот пример в вашем браузере, то заметите, что теги HTML в браузере не отображаются. Они читаются только браузером, чтобы знать, какой тип контента вы написали.

Самозакрывающиеся элементы

Некоторые элементы не содержат закрывающего тега и такие элементы называются самозакрывающими. Естественно, у них нет и содержимого.

Такие элементы просто получают содержимое или поведение через атрибуты в пределах одного тега.

Например, тег изображения. Содержимое элемента <img> в примере присваивается с помощью атрибута src и значения в кавычках, которое сообщает браузеру путь, откуда взять картинку. Дополнитнльный атрибут alt хранит в себе описание картинки.

<img src="https://webref.ru/assets/images/html-tutorial/element.png" alt="Картинка">

К типичным самозакрывающимся элементам относятся:

  • <meta>
  • <link>
  • <img>
  • <source>
  • <input>
  • <hr>
  • <br>
  • <wbr>
  • <embed>
  • <param>

Вложения элементов

Одни элементы допустимо вкладывать внутрь других, однако следует соблюдать их порядок. На рисунке демонстрируется, как можно и нельзя добавлять один элемент внутрь другого.

Если связать открывающие и закрывающие теги между собой скобкой, как показано на рисунке, то несколько скобок не должны пересекаться между собой. Любое пересечение условных скобок говорит о том, что правильная последовательность тегов нарушена.

Атрибуты

Атрибуты действуют как дополнительная информация, привязанная к элементу HTML, и предоставляют ему дополнительные свойства. Атрибуты пишутся внутри тега HTML. Они также не отображаются в браузере.

В данном случае, class это имя атрибута, а editor-note это значение атрибута. Класс позволяет дать элементу идентификационное имя, которое может позже использоваться, чтобы обращаться к элементу с информацией о стиле и прочих вещах.

Атрибут всегда должен иметь:

  1. Пробел между ним и именем элемента (или предыдущим атрибутом, если элемент уже имеет один или несколько атрибутов).
  2. Имя атрибута, за которым следует знак равенства.
  3. Значение атрибута, заключённое с двух сторон в кавычки.

Наиболее распространённые атрибуты:

  • атрибут id - идентифицирует элемент;
  • атрибут class - классифицирует элемент;
  • атрибут src - определяет источник встраиваемого содержимого;
  • атрибут href - указывает ссылку на связанный ресурс.

Резюме.

Атрибуты определяются в открывающем теге после имени элемента. В общем, атрибуты включают в себя имя и значение. Формат для этих атрибутов состоит из имени атрибута со знаком равенства за ним, а затем в кавычках идёт значение атрибута.

Элемент ссылки объявлен с помощью открывающего тега <a> и закрывающего тега </a> охватывающих текст, а также атрибута и значения адреса ссылки объявленной через href="http://shayhowe.com" в открывающем теге.

Комментарии

<!-- -->

Комментарии в коде HTML-документа предназначены для пояснения участка кода. Это улучшает читабельность кода и упрощает процесс редактирования в дальнейшем.

Комментарии игнорируются браузером и не отображаются на странице пользователя. Однако их можно увидеть в исходном коде веб-страницы

Синтаксис

<!-- текст комментария -->

Закрывающий тег

Обязателен

Комментарии, также используются для технических целей, когда требуется временно исключить участок кода из верстки.

Чтобы быстро закомментировать часть кода, нужно выделить необходимый участок и зажать клавиши [CTRL] + [ / ]

Чтобы добавить закомментированный участок в код, нужно его раскомментировать. Опять выделяем закомментированный участок и зажимаем [CTRL] + [ / ]

Валидация кода

Независимо от того, насколько аккуратно мы пишем наш код, ошибки неизбежны. К счастью, при написании HTML и CSS у нас есть валидаторы для проверки нашей работы. Сервис W3C предлагает валидаторы HTML и CSS, которые сканируют код на ошибки. Проверка нашего кода не только помогает правильно отображать его во всех браузерах, но и помогает обучению передовому опыту при написании кода.
